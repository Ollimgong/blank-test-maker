# ADR-002: BTM 마크업 스펙과 Transparent 숨김 전략

**작성/업데이트**: 2026-04-10  
**상태**: 채택 (Accepted)

## 배경 (Context)
교사(작성자)가 입력하는 텍스트는 "정답지(ANSWER)" 모드에서는 전부 보이지만, "시험지(WORKSHEET)" 모드에서는 특정 부분을 블라인드(빈칸) 처리해야 합니다. 어떤 영역을 블라인드할 것인지 직관적으로 선언하는 마크업이 필요했습니다.

## 결정 (Decision)
1. **문법적 선언**: 텍스트 상자 내의 모든 요소는 BTM(_Blank Test Markup_) 인라인 규칙을 파서를 통해 평가받습니다.
    - `@태그`와 `[라벨]`, 헤더 류는 시각적 UI 뱃지로 렌더링합니다.
    - `!` 기호가 포함된 줄은 "시험지에 표시되어야 하는 문제 텍스트"로 평가받습니다(`vis: true`).
2. **Transparent Masking 채택**:
    - 일반 텍스트 라인은 시험지 모드에서 DOM 요소 자체를 지우는(Remove) 방식이 아니라, 색상을 `transparent`(투명화)하는 정책을 사용합니다.
    - 이유: A4 용지의 표 칸 높이와 줄바꿈 위치가 **정답지나 시험지 양쪽 모두 1px의 오차도 없이 동일한 위치를 유지**해야 하기 때문입니다. 문자열 자체를 삭제해버리면 셀의 높이가 붕괴됩니다.

## 영향 (Consequences)
- 레이아웃이 무너지지 않는 안정적인 양방향 출력(A/W 모드) 환경이 조성되었습니다.
- 단, 브라우저상에서 사용자가 투명 텍스트를 드래그하면 원래 정답을 볼 수 있다는 클라이언트-사이드 취약점이 존재합니다. 다만 본 앱의 사용자는 '인쇄'를 목적으로 하는 교사이므로 이는 심각한 보안 이슈로 취급하지 않습니다.
