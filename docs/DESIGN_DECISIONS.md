# 설계 결정사항

이 문서는 개발 과정에서 내린 주요 설계 결정과 그 이유를 기록합니다.
Claude Code에서 작업을 이어갈 때 맥락 파악용.

---

## 왜 React 단일 파일인가?

- 사용자 요구: "서버 없이 HTML 링크로" 사용
- Claude.ai Artifact 시스템에서 .jsx 파일을 바로 렌더링 가능
- 외부 라이브러리 의존 최소화 (React 기본 훅만 사용)
- 향후 standalone HTML로 빌드 가능성 열어둠

## 왜 인라인 스타일인가?

- Tailwind CSS를 사용할 수 있지만, pre-defined 유틸리티 클래스만 사용 가능
- 동적 스타일링(태그 색상, 조건부 배경 등)이 많아서 인라인이 더 직관적
- CSS 파일 분리 없이 단일 파일 유지

## 왼쪽/오른쪽 독립 구조

- 초기에는 하나의 "행"이 왼쪽+오른쪽을 묶는 구조
- 사용자 피드백: "분리된 섹션이기 때문에 따로 생각하는 게 좋겠어"
- 변경: 셀 배경색 독립 판단, 이동(▲▼)도 각 칸 독립
- **데이터 구조는 유지**: 한 row에 l/r이 묶여있되, 조작은 독립

## 헤더를 셀 속성(hdr)으로 변경

- 기존: 단원 레벨의 `headerL`/`headerR`로 고정 헤더 행
- 변경: 각 셀의 `hdr` 속성으로 어느 행에서든 헤더 스타일 적용 가능
- 이유: 부사 파일처럼 PRACTICE 없이 SUMMARY만 있는 경우, 중간에 새 섹션을 시작하는 경우 등 유연한 레이아웃 필요
- 왼쪽/오른쪽 독립: 같은 행에서 왼쪽만 헤더, 오른쪽은 일반 셀 가능
- 테두리도 셀별 독립 계산: hdr↔일반 경계에서만 굵은 초록 테두리

## A4 인쇄 최적화 — 행 높이 고정

- 사용자: "노트 같은 느낌을 살리려고 한줄씩 입력하는 거"
- 행이 늘어나면 A4 전체 크기가 달라짐 → 인쇄 품질 저하
- 결정: ROW_H = 27px 고정, overflow: hidden, text-overflow: ellipsis
- 긴 내용은 편집기에서 확인 가능, 인쇄물에서는 잘림

## 30행 고정 — 노트 느낌

- A4 인쇄 시 30행이 딱 맞는 크기
- 행 추가/삭제 기능 제거 — 고정된 노트 느낌 유지
- 데이터에도 항상 30행 (padRows로 자동 패딩)
- 기존 삭제 2단계 확인도 함께 제거됨

## 시험지 빈칸 처리

- 초기: 빈칸 밑줄 표시 (_______)
- 사용자 요청: "시험지에 빈칸은 없었으면 좋겠어"
- 변경: ans=true인 셀은 시험지에서 내용만 숨기고 빈 공간만 남김
- 구조(태그, 번호, 라벨)는 유지

## 태그 색상 선정 기준

- 원본 PDF의 색상 참고하되, 더 선명하고 구분되게 조정
- 개념(파랑)과 예문(초록): 가장 자주 사용 → 차분하면서 구분 명확
- 주의(핑크레드)와 종류(빨강): 경고/강조 의미 → 붉은 계열이지만 구분
- 형태 계열(앰버/오렌지): 형태1~3까지 유사색 통일

## 대화 히스토리 요약

### 1차: 초기 구현
- 5개 참고 PDF 분석 → 공통 구조 파악
- SUMMARY + PRACTICE 2단 테이블 구조 결정
- "과목" 단위, 답지/시험지 토글, 인쇄 기능

### 2차: 대폭 개선
- "과목" → "단원" 명칭 변경
- 그룹화 기능 추가
- 편집+미리보기 분할 화면
- 태그를 컬러 배지 팝업으로 시각화
- 시험지 빈칸 제거 (빈 공간만)
- 로고 업로드 + 이름란

### 3차: 세부 수정
- 태그 팝업 overflow 이슈 수정
- 셀 배경색 독립 처리
- 섹션 헤더 편집 가능
- A4 인쇄 최적화 (행 높이 고정)
- 태그 색상 정제

### 4차: 독립 이동 + 안전장치
- 왼쪽/오른쪽 독립 ▲▼ 이동
- 삭제 2단계 확인

### 5차: Vite 개발환경 + 대폭 개선
- Vite + React 로컬 개발환경 구축 (핫 리로드)
- localStorage 폴리필 (window.storage 호환)
- A4 인쇄 최적화 (1페이지 출력, 미리보기=인쇄 동일 레이아웃)
- 헤더를 셀 속성(hdr)으로 변경 — 유연한 섹션 헤더 배치
- 30행 고정 노트 레이아웃, 행 추가/삭제 제거
- 좌우 셀 테두리 완전 독립 (hdr 경계 각각 계산)
- Pretendard 웹폰트 기본 적용 + 커스텀 폰트 업로드
- 학원이름 → 슬로건, 푸터: 슬로건 왼쪽 / 로고 오른쪽
- 이름/날짜 기입란 제거
- 기존 데이터 자동 마이그레이션 (headerL/R → hdr, academyName → slogan)
